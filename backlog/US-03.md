# US-03 · Grounded answers with source citations

**Priority:** P0 · **Story points:** 2

## Story

> As an investor, I want the assistant to cite the sources behind its answers so that I can verify the information against the original materials.

## Acceptance criteria (verbatim from `Candidate_04_User_Stories.docx`)

1. Every substantive claim in an answer references the source document it came from.
2. I can distinguish which parts of an answer came from which document when multiple sources are used.
3. The assistant does not present information as though it came from a document when it did not.
4. Source references include enough detail (document title, type, or date) for me to find the original material.

## Why this matters

The PRD NFR on grounding is absolute: "Every substantive claim in an answer must be traceable to a specific source document or data point." This story turns the ingest metadata from US-02 into honest, inline citations an investor can verify.

## Definition of done

- `core/assistant.py` implements `answer(question, history) -> AnswerWithCitations` using the Anthropic tool-use loop with a `search_documents` tool backed by `core/retrieval.py`.
- The system prompt in `core/llm.py` requires the model to cite inline (e.g. `[1]`, `[2]`) and to only cite from tool results — never from prior knowledge.
- The returned `AnswerWithCitations.citations` list carries `Citation` objects whose fields come directly from the retrieved chunks. The model is not allowed to rewrite titles, periods, or page numbers.
- When multiple documents are used in one answer, each inline marker maps to a distinct citation (US-03 AC2).
- When retrieval returns nothing relevant, the assistant says so and returns an empty citations list rather than inventing one.

## Verification

```bash
source .venv/bin/activate
uv run python -c "
from core.assistant import answer
r = answer('What did management say about revenue in the FY24 annual report?', [])
print(r.text)
print('---')
for i, c in enumerate(r.citations, 1):
    print(i, c.doc_title, c.doc_type, c.period, 'p.'+str(c.page))
"
```

Confirm:

1. The answer text contains at least one inline marker and the marker count matches the citations list length.
2. Each citation resolves to a real file under `data/pdfs/` — the title, type, period, and page number are accurate against the source PDF.
3. Asking something clearly outside the corpus (e.g. "What did the CFO say in FY19?") returns a refusal with an empty citations list, not a hallucinated source.
