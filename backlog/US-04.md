# US-04 · Market data answers

**Priority:** P0 · **Story points:** 1

## Story

> As an investor, I want to ask questions about OML share price history so that I can understand market data alongside the company's published materials.

## Acceptance criteria (verbatim from `Candidate_04_User_Stories.docx`)

1. The assistant can answer questions about historical OML share price data, such as closing price on a given date or price movement over a period.
2. Answers include the specific dates, prices, or ranges used.
3. If market data is temporarily unavailable, the assistant says so rather than guessing or making up numbers.
4. Market data answers are clearly distinguished from document-sourced answers.

## Why this matters

Market data is the structured complement to narrative PDFs. It is also the foundation for US-05 (combined answers). The Solution Design's free-tier guidance for Marketstack (`v1`/HTTP/`OML.XASX`) is stale — the **v2** endpoint works over **HTTPS** with symbol **`OML.AX`** on the free tier and is what we use. Alpha Vantage's free tier is 25 requests/day **and does not cover ASX listings**, so it remains a scaffold behind the same interface. Caching every call to disk is non-negotiable.

## Definition of done

- `core/prices.py` defines a `PriceProvider` interface and two concrete implementations: `MarketstackProvider` (v2 endpoint over HTTPS, symbol `OML.AX`) and `AlphaVantageProvider` (symbol `OML.AX`, kept as a scaffold — free tier does not cover ASX listings; see `DECISIONS.md`).
- A factory selects the provider based on env (`MARKETSTACK_API_KEY` / `ALPHAVANTAGE_API_KEY`). **Marketstack is primary** for OML; Alpha Vantage is the secondary scaffold. If neither key is set, the tool returns an explicit unavailable result.
- Every API response is cached as JSON under `data/cache/` keyed by provider + symbol + function + params. Reads check cache first, always.
- `get_price_history(start, end)` is exposed to the Anthropic tool-use loop in `core/assistant.py` as a second tool.
- Market-data citations use `doc_type="market_data"` and populate `period` as the ISO date range; `page` is `None`; `snippet` contains the specific prices or series used.
- When the underlying API fails and no cache is available, the tool returns an explicit "unavailable" result and the assistant surfaces that to the user (US-04 AC3).
