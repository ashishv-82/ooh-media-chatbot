# US-07 · Reusable assistant capability

**Priority:** P1 · **Story points:** 2 · **Stretch goal**

## Story

> As an internal business user, I want to access the same investor assistant from tools I already use, such as Claude Desktop, so that the capability is not locked to a single web page.

## Acceptance criteria (verbatim from `Candidate_04_User_Stories.docx`)

1. The same assistant capability that powers the web chat is available through at least one other interface or tool.
2. A question asked through the second interface returns the same quality of answer as the web chat.
3. The underlying capability is shared, not rebuilt separately for each surface.
4. A business user can set up and use the second interface with clear documentation or instructions.
5. The approach is extensible. Adding another surface in the future should reuse the existing capability rather than requiring a new implementation.

## Why this matters

The brief explicitly name-checks Claude Desktop, and the Solution Design's headline design stance is "Reusable capability, multiple surfaces, grounded answers." This story proves the `core/` module really is a reusable capability and not just a private helper for the Streamlit app. It is a P1 stretch goal - build it only if US-01 through US-06 are complete and stable.

## Definition of done

- `mcp_server.py` at the repo root is a stdio MCP server built on the official `mcp` Python SDK.
- It exposes exactly one tool, `ask_oohmedia_investor_chat(question: str)`, whose implementation is a thin wrapper around `core.assistant.answer()`.
- The server imports only from `core` - no direct calls to `anthropic`, `openai`, `chromadb`, or any market-data SDK.
- All logging goes to stderr. Nothing is written to stdout except MCP protocol traffic.
- `README.md` contains a "Claude Desktop setup" section with the exact JSON snippet to add to `claude_desktop_config.json`, including the command, args, and required env vars.
- Answers returned via MCP carry the same `AnswerWithCitations` content as the web chat for the same question.

## Verification

```bash
source .venv/bin/activate
uv run python mcp_server.py   # should start and wait on stdio, no stdout chatter
```

Then:

1. Add the server to `claude_desktop_config.json` using the README snippet and restart Claude Desktop.
2. In Claude Desktop, ask the same showcase question from US-05 and confirm the answer cites the same documents and price points as the Streamlit app.
3. Confirm the MCP server is under ~50 lines of real logic - if it is larger, reasoning has leaked out of `core/` and must be pulled back in.
