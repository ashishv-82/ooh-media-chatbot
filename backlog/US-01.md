# US-01 · Chat interface for investor questions

**Priority:** P0 · **Story points:** 2

## Story

> As an investor, I want a web-based chat interface where I can ask questions about oOh!media and see the assistant's answers, so that I can interact with investor materials conversationally.

## Acceptance criteria (verbatim from `Candidate_04_User_Stories.docx`)

1. The interface opens in a browser and is ready to accept a question.
2. I can type a question and receive a visible response.
3. The current session's conversation history remains visible on screen.
4. If something goes wrong behind the scenes, I see a clear error message rather than a blank screen or a hung request.
5. I can ask follow-up questions within the same session.

## Why this matters

This is the primary surface an investor touches. It is also the place where `core.assistant.answer()` is first exercised end-to-end against real data. Nothing about reasoning, retrieval, or citations lives here — this story is about a clean, honest chat shell that fails visibly when the backend fails.

## Definition of done

- A Streamlit app at `app.py` can be launched with `streamlit run app.py` and opens a chat interface in the browser.
- The app holds conversation history in `st.session_state` for the life of the session and renders prior turns above the input.
- User input is passed to `core.assistant.answer(question, history)` and the response (text + citations) is rendered in the chat.
- Backend exceptions are caught and shown via `st.error(...)` with the exception message — never a blank screen or a hung spinner.
- Citations are rendered from the `AnswerWithCitations` structure returned by `core.assistant.answer()`. Do not reformat or invent citation fields in the UI.
- `app.py` imports only from `core`. It does not import `openai`, `anthropic`, `chromadb`, or any market-data SDK.

## Verification

Run from the repo root with the venv active:

```bash
source .venv/bin/activate
streamlit run app.py
```

Then in the browser:

1. Confirm the page loads with an empty chat and a visible input box.
2. Ask a simple question (e.g. "What's in the FY24 annual report about revenue?") and confirm a streamed response with at least one citation.
3. Ask a follow-up (e.g. "And how did the share price react?") and confirm the prior turn is still visible.
4. Temporarily break the backend (e.g. unset `ANTHROPIC_API_KEY`), re-ask, and confirm a clear red error message appears instead of a hang.
